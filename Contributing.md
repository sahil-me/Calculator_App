# Contributing to Calculator App

Contributions of all kinds are welcome. Whether you're fixing bugs, improving the user interface, enhancing functionality, optimizing the code, or improving the documentation, every contribution helps make this project better.

If you find this project useful, consider sharing it with others, referencing it in your projects, or giving the repository a ⭐ to support its development.

Please remember to follow our **[Code of Conduct](https://github.com/sahil-me/Calculator_App/blob/main/CODE_OF_CONDUCT.md)** and maintain a respectful and collaborative environment for everyone.

---

## Ways to Contribute

There are many ways you can contribute to **Calculator App**.

* **Feature Development** – Add new features such as scientific calculations, keyboard shortcuts, history, themes, memory functions, or additional mathematical operations.
* **Bug Fixes** – Identify and resolve issues in the application.
* **UI Improvements** – Enhance the application's design, usability, accessibility, or responsiveness.
* **Performance Optimization** – Improve the application's efficiency and code quality.
* **Documentation** – Improve the README, user guides, or project documentation.
* **Testing** – Add or improve test cases to ensure application reliability.

> Every contribution, no matter how small, is greatly appreciated. ❤️

---

## Reporting Bugs

If you discover a bug, please open an issue and include as much information as possible.

Helpful information includes:

* Operating System and version
* Java version
* Steps to reproduce the issue
* Expected behavior
* Actual behavior
* Screenshots (if applicable)
* Complete error messages or stack traces

Before creating a new issue, please check whether it has already been reported.

---

## Requesting a Feature

Have an idea to improve the **Calculator App**?

Open an issue and describe:

1. The problem your feature solves.
2. Your proposed solution.
3. How the feature should work.
4. Screenshots, or code examples.

Well-described feature requests help us review and implement improvements more efficiently.

---

## Improving Documentation

Documentation improvements are always welcome.

Examples include:

* Fixing typos
* Improving explanations
* Updating screenshots
* Adding installation or usage instructions
* Enhancing code comments

## Fixing outstanding issues

If you notice an issue with the existing code and have a fix in mind, feel free to **[start contributing](https://docs.github.com/en/pull-requests/collaborating-with-pull-requests/proposing-changes-to-your-work-with-pull-requests/creating-a-pull-request)** and open
a Pull Request!

### Making Code Contributions

<details>

1. **Fork the Repository**
    - Go to the GitHub repository and click on the "Fork" button.

2. **Clone Your Fork**
    ```bash
    git clone https://github.com/<your-username>/Calculator_App.git
    ```

    ```bash
    cd Calculator_App
    ```

3. **Create a New Branch**
    ```bash
    git checkout -b feature/your-feature-name
    ```

4. **Make Your Changes**
    - Write clean and well-structured code.
    - Follow the existing project style.
    - Test your changes before committing.

5. **Commit Your Changes**
    ```bash
    git add .
    git commit -m "Describe your changes"
    ```

6. **Push Your Branch**
    ```bash
    git push origin feature/your-feature-name
    ```

7. **Create a Pull Request**

    Submit a Pull Request with a clear description of:
      - What you changed
      - Why the change was made
      - Screenshots (if the UI was modified)

8. **Respond to Feedback**
    - If reviewers suggest improvements, update your Pull Request accordingly.

</details>

This project welcomes contributors who are interested in helping maintain and improve the application.

Contributors who consistently provide high-quality improvements, review pull requests, improve documentation, and actively participate in the project may be invited to become maintainers.

We appreciate every contribution and look forward to collaborating with you! 😊
